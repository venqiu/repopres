    决策指南：保存资源与目的地

    根据你的身份、可用资源和技术能力，你可以选择不同的路径来保存软件仓库：

    • 完全由自己在本地保存；
    • 把源代码交给像 Software Heritage 这样的公共软件档案库；
    • 或者与某个机构的数字保存系统合作。

    每一种方案都有优点与局限，你可以根据下面的特点作出选择。

    本地保存路线#

    • Pros 优点：

      • 只需要依赖你自己，不需要等待外部机构审批。
      • 你可以自己制定保存策略： 例如文件夹结构、想要保存哪些相关材料（文档、截图、论文等）、需要记录哪些说明信息。
      • 隐私控制完全在你手里： 你可以决定谁能访问这些软件、副本存放在哪里。

    • Cons 缺点：

      • …（此处原文用省略号表示，具体劣势可根据你的实际情况补充）

    Software Heritage Route（Software Heritage 路线）#

    • Pros 优点：

      • 有机构背书的长期项目，并且有稳定的资金与持续运营。
      • 归档流程严谨、集中管理，适合长期保存。
      • 你的软件会与其他项目放在一起，被放入更大的历史语境中——这对软件史研究非常有价值。

    • Cons 缺点：

      • 受欧洲版权法（European copyright law）约束。
      • 元数据字段由 Software Heritage 定义，用户无法随意更改。
      • 归档决策需要经过审核，某些请求可能不会被接受。

    Institutional Route（机构路线）#

    • Pros 优点：

      • 机构层面的资源与稳定性（经费、硬件、长期规划等）。
      • 可以与数字保存人员（digital preservationists）及其他同事合作，为软件提供更完整的文献与上下文。

    • Cons 缺点：

      • 需要使用你所在机构已经部署的 DAMS（Digital Asset Management System）， 而这些系统未必是专门为软件归档设计的，有时会不太适配源代码和版本控制的特性。
